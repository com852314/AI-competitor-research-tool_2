# 競品調研 企劃操作手冊

**適用對象**：負責每週競品調研的企劃同仁
**最後更新**：2026-05-25
**前置條件**：需要能登入 PT / BP / CP / TCG / BAJI 五個平台

---

## 整體流程概覽

```
【每週執行】

  你（企劃）          Cowork Session           Claude Code
  ──────────         ──────────────           ────────────
  登入 5 個平台
       │
  複製 DATA_CAPTURE
  _PROMPT.md 內容
  開新 cowork 對話
       │
       ├─ 給第一批資料 ──→ 辨識 PT/BP/CP
       │                   整理排名
       ├─ 給第二批資料 ──→ 辨識 TCG/BAJI
       │                   整理排名+新遊戲
       │
       │        ←── 輸出 raw_input.json
       │
  複製 JSON 內容
  貼給 Claude Code ──────────────────→ 執行 capture_week.py
                                        回報新廠家/待辨識
       │        ←────────────────────
  確認新廠家名稱
  提供待辨識名稱 ──────────────────→ 補進主表
                                        git commit + push
       │        ←────────────────────
  GitHub Pages
  更新完成 ✓
```

預計總時間：**30–60 分鐘**（視辨識困難度）

## 需要用到的工具

| 工具 | 用途 | 在哪裡 |
|---|---|---|
| Claude Cowork | 辨識平台資料、輸出 JSON | claude.ai 開新對話 |
| Claude Code | 執行腳本、更新主表、push | 本專案的 Code 對話 |
| 瀏覽器 | 登入 5 個平台查看排行 | 你自己的瀏覽器 |
| `docs/DATA_CAPTURE_PROMPT.md` | 給 cowork 的任務說明 | 本專案 docs 資料夾 |

## 第一階段：準備資料（你 + Cowork）

### Step 1｜開啟 Cowork Session

1. 打開 [claude.ai](https://claude.ai)，**開一個全新的對話**
2. 打開專案中的 `docs/DATA_CAPTURE_PROMPT.md`
3. 把整份文件內容**複製貼上**作為第一則訊息送出
4. 等 cowork 回覆「已了解，請提供第一批資料」後繼續

> ⚠️ 每週都要開**新對話**，不要沿用上週的 cowork session，避免 context 混亂。

---

### Step 2｜第一批：PT + BP + CP

依序提供三個平台的資料給 cowork：

**PT（PT Gaming）**
- 登入後截圖 Slot 熱門排行頁（Top 20）
- 把截圖直接貼進 cowork 對話
- 等 cowork 整理完，確認有 20 筆再繼續

**BP（BingoPlus）**
- 登入後開啟 Slot 排行頁
- 選取整個排行清單的 HTML 或把排名文字貼進 cowork
- 等 cowork 整理完，確認有 20 筆

**CP（Casino Plus）**
- 登入後開啟 Slot 排行頁
- 貼文字或截圖給 cowork
- CP **沒有廠家資訊**，cowork 會自動用對照表填入，查不到的填「(待補)」，這是正常的

> 確認三個平台都有 20 筆後，告訴 cowork「第一批確認，繼續第二批」

---

### Step 3｜第二批：TCG + BAJI

**TCG（Luckyi）**
需要提供兩份資料：
1. Slot 熱門排行 Top 20
2. NEW 分類的新遊戲清單（**最多前 20 筆**，超過不用給）

**BAJI（Baji）**
需要提供兩份資料：
1. Recommend 熱門排行 Top 20
2. New-Old 排序的新遊戲清單（**最多前 20 筆**）

> **給資料前請先確認頁面正確**：例如要給 TCG NEW 分類，確認畫面顯示的是「NEW」而非「Recommend」或「Popular」。如果頁面不對，先切換好再截圖/複製給 cowork。

> Cowork 可以在頁面內滾動讀取資料，但不會自行切換分頁。如果 cowork 回報「頁面內容與預期不符」，請先確認後再繼續。

> 新遊戲可能有只看到 ID 沒有名稱的，填「(待辨識)」是正常的，之後可補名。

---

### Step 4｜取得 raw_input.json

所有資料確認後，告訴 cowork：

> 「全部確認，請輸出完整的 raw_input.json」

cowork 會輸出一個 JSON 代碼區塊，例如：

```json
{
  "captureDate": "2026-05-XX",
  "platforms": {
    "PT": { "rankings": [...] },
    "BP": { "rankings": [...] },
    "CP": { "rankings": [...] },
    "TCG": { "rankings": [...], "newGames": [...] },
    "BAJI": { "rankings": [...], "newGames": [...] }
  }
}
```

**複製整個 JSON 內容（包含 `{` 到最後的 `}`）**，準備交給 Claude Code。

---

## 第二階段：執行腳本（Claude Code）

### Step 5｜把 JSON 交給 Claude Code

打開 Claude Code 的對話，說：

> 「這是本週的 raw_input.json，幫我跑流程：
> （貼上 JSON 內容）」

Claude Code 會自動：
1. 把 JSON 存入 `data/incoming/raw_input.json`
2. 執行 `scripts/capture_week.py`
3. 回報執行結果

---

### Step 6｜處理 Claude Code 的回報

腳本執行完後，Claude Code 會回報三種情況：

#### 情況 A：有新廠家

```
⚠ 發現 2 個新廠家（需確認後補進主表）：
   - NetEnt
   - KingMidas
```

你需要確認廠家名稱是否正確，告訴 Claude Code：
> 「廠家名稱都對，幫我補進去」
> 或
> 「NetEnt 的正確名稱是 NetEnt，KingMidas 沒問題，可以補」

#### 情況 B：有待辨識遊戲

```
ℹ 以下遊戲名稱待確認（有 gameId，需查名稱）：
   - TCG：PP0992（Pragmatic Play）
   - BAJI：PG-SLOT-058（PG Soft）
```

你需要查明遊戲名稱後告訴 Claude Code：
> 「PP0992 是 Gates of Olympus Super Scatter，PG-SLOT-058 是 Mahjong Ways 2」

Claude Code 會自動補名並更新資料。

#### 情況 C：都沒問題

直接跳到 Step 7。

---

### Step 7｜確認並 Push

Claude Code 完成補名/補廠家後，告訴它：

> 「沒問題，可以 commit 跟 push」

Claude Code 會執行：
```
git add data/weekly/ data/master/ data/manifest.json
git commit -m "data: W{N} 快照 (YYYY-MM-DD)"
git push origin main
```

---

## 第三階段：驗收

### Step 8｜確認 GitHub Pages 更新

Push 後約 **1–2 分鐘**，打開 Viewer 網頁：

```
https://com852314.github.io/AI-competitor-research-tool_2/
```

確認清單：
- [ ] 週次下拉選單出現最新的 W（本週）
- [ ] 📊 本週快照：5 個平台都有資料
- [ ] 📈 跨週趨勢：折線圖、常勝軍、進出榜都有更新
- [ ] 沒有「(待辨識)」大量出現（偶爾 1–2 筆可接受）

如果頁面還是舊的 → 按 `Ctrl + F5` 強制重整。

## 例外狀況處理

### 某個平台當週無法取得資料

告訴 cowork 略過該平台，在 raw_input.json 中那個平台的 rankings 給空陣列 `[]`。
腳本執行時會印出警告但不影響其他平台。

### Cowork 輸出的 JSON 格式有問題

複製 JSON 後貼給 Claude Code 時說：
> 「這份 JSON 貼過來後格式可能不完整，幫我先驗證一下再執行」

Claude Code 會先檢查格式再繼續。

### 新遊戲特別多，待辨識款數很多

先讓 Claude Code 跑腳本，名稱填 `(待辨識)` 先進資料庫。
當週先 push，之後查到名稱再告訴 Claude Code 補名（說「幫我補 WN 的待辨識遊戲名稱」）。

### 同一款遊戲在不同平台名稱不同

這是正常的。例如「Fortune Gems 3」在 BAJI 叫「Fortune Gems III」。
正規化名稱以**主表的 normalizedName 為準**，其他平台填 rawVendor/name 照平台原樣。

## 檔案說明（接手者快速查找）

```
競品調研(排名)/
│
├── index.html               ← Viewer 本體（不需手動改）
├── docs/
│   ├── DATA_CAPTURE_PROMPT.md  ← 每週給 cowork 的提示詞（複製貼上用）
│   ├── 企劃操作手冊.md          ← 本文件
│   └── 使用手冊.md             ← 給報告瀏覽者看的說明
│
├── data/
│   ├── manifest.json           ← 自動更新，不要手動改
│   ├── weekly/                 ← 每週快照（W1, W2, W3...）
│   ├── master/
│   │   ├── games.json          ← 遊戲主表（腳本自動維護）
│   │   ├── vendors.json        ← 廠家主表（新廠家需手動確認）
│   │   └── platforms.json      ← 平台設定（不需常改）
│   └── incoming/
│       └── raw_input.json      ← 每週暫存（cowork 輸出的資料）
│
└── scripts/
    └── capture_week.py         ← 每週執行的主腳本
```
