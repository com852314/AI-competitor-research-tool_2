## 一、專案概述

### 任務
追蹤 5 個競品平台的 Slot 遊戲熱門榜與新遊戲，每週執行一次，累積後做跨週趨勢與跨平台分析。

### 5 個調研平台
| 代號 | 全名 | 市場 | 抓取方式 |
|---|---|---|---|
| PT | PT Gaming | 菲律賓 | 截圖 + AI 視覺辨識 |
| BP | BingoPlus | 菲律賓 | HTML 直接 JS 抓 |
| CP | Casino Plus | 菲律賓 | HTML（廠家需從主表查詢）|
| TCG | Luckyi | 菲律賓 | HTML + 名稱視覺辨識首次快取 |
| BAJI | Baji | 孟加拉 | HTML + 名稱視覺辨識首次快取 |

OKB 已從調研範圍移除。

### 每週調研項目
- 各平台 Slot 分類 Top 20 熱門遊戲（含名稱、廠家、排名）
- TCG NEW 分類新遊戲、BAJI New-Old 排序新遊戲

## 五、腳本規格

### `capture_week.py`（每週執行、已實作）
**輸入**：`data/incoming/raw_input.json`（AI 整理）

**動作**：
1. 從 weekly/ 找最大 W 編號 +1
2. 從 games.json 找主表 ID（用 name+vendor 配對）
3. 新遊戲自動加進 games.json
4. 新廠家報告給用戶（不自動加）
5. 對比上週計算 isNewToTop20、lastWeekRank、rankChange
6. 寫 weekly/W{N}_{date}.json
7. **自動更新 manifest.json**（給 GitHub Pages 用）
8. 更新 platforms.json 的 lastCapture

**用法**：
```bash
python scripts/capture_week.py data/incoming/raw_input.json
```

### `index.html`（viewer 本體、根目錄、已實作）
**雙模式**：
- 本機 file:// 開啟 → 顯示「📂 選擇 data 資料夾」按鈕，手動載入
- 網站 https:// 開啟 → **自動 fetch ./data/manifest.json + 所有檔案**

**頁籤**：
1. ✅ 📊 本週快照 - 5 平台 Top 20 並排、★ 跨平台款標示、點擊展開備註
2. ✅ 🌐 跨平台分析 - ≥3 平台款表格、廠家市占長條圖、自動觀察
3. ✅ 📋 主表瀏覽 - 22+ 款、可搜尋、可按平台數篩選
4. ✅ 🏢 廠家對照 - 26+ 廠家、各平台代碼對照
5. ⏳ 📈 跨週趨勢 - Phase 2E（需 ≥2 週資料）

## 六、待實作功能

### ⏳ Phase 2D：新遊戲頁籤（優先實作）

**背景**：原本 Excel 有 `TCG_新遊戲`、`BAJI_新遊戲`、`整合_新遊戲_菲律賓`、`整合_新遊戲_孟加拉` 4 張工作表。JSON 結構也保留了 `platforms.TCG.newGames` 跟 `platforms.BAJI.newGames`。但 **index.html viewer 目前沒有對應的頁籤顯示**、這份資料看不到。

**新增頁籤位置**：`📰 新遊戲`（介於「跨平台分析」與「主表瀏覽」之間）

**內容**：

1. **本週新遊戲清單**（按市場分區）
   - **菲律賓區**：TCG NEW 分類所有遊戲（含 order、name、vendor、gameId、note）
   - **孟加拉區**：BAJI New-Old 排序所有遊戲（同上）
   - 廠家用 vendor-tag 上色（同其他頁籤）

2. **跨平台同款偵測**（亮點）
   - 對每款新遊戲查主表 ID
   - 標示「此款也出現在其他平台」的遊戲
   - 例如：TCG 新遊戲「LUCKY JAGUAR 500」與 BAJI 新遊戲「Lucky Jaguar 500」是同一款（主表配對）

3. **主表配對狀態**
   - 已在主表 → 標 ✓ + 主表 ID
   - 未在主表 → 標「未配對」灰色

4. **跨週分析**（W2 起）
   - 「上週新遊戲、本週是否進熱門榜」連結
   - 「連續 N 週都在 NEW 清單」持久新遊戲統計

5. **搜尋與篩選**
   - 搜尋遊戲名 / 廠家
   - 篩選：本週新 / 累積（多週）

**實作工作量預估**：類似 Phase 2B + 2C，約 1 個批次（中等規模）

## 七、給未來 AI 的接手指引

### 如果用戶說「我們繼續這個專案」

1. **先讀本文件**（如果你在讀這段，已經做了）
2. **看資料夾結構**：

3. **看 `docs/週執行SOP.docx` v4** 是當週執行步驟的權威來源
4. **不要重做已完成的事**：W1 已完成、主表已建立

### 待辦項目（按優先順序）

1. **Phase 2D 新遊戲頁籤**（第六章規格）→ W1 就能用、優先做
2. **W2 抓取**（用戶執行）→ 累積資料
3. **Phase 2E 跨週趨勢**（W2 後）→ 折線圖、常勝軍等

### 每週執行的標準流程（SOP v4）

```
0. 用戶登入 5 平台 → 提供分頁
1. AI 抓 PT + BP + CP → 暫存 raw_input.json 的對應欄位
   → 回報辨識失敗清單給用戶 → 用戶確認 → 修正
2. AI 抓 TCG + BAJI（rankings + newGames）→ 補進 raw_input.json
   → 回報辨識失敗、新廠家、跨平台同款建議 → 用戶確認 → 修正
3. AI 執行：python scripts/capture_week.py data/incoming/raw_input.json
   → 自動寫入 weekly/W{N}_*.json + 更新 master + 更新 manifest.json
4. AI 回報：新增的主表 ID、發現的新廠家、≥3 平台款摘要、本週觀察
5. 用戶 git push → GitHub Pages 自動更新、同事打網址即可看
```

### 用戶想看資料
- **首選**：請用戶開 GitHub Pages 網址（最方便）
- **次選**：請用戶開 index.html 本機檔（雙擊）
- **避免**：不再生成 Excel

### 用戶想編資料
- 主表 / 廠家 / 平台對照在 `data/master/*.json`
- 用戶可直接編 JSON，或在對話中告訴 AI 怎麼改
- 改完驗證：`python -m json.tool data/master/games.json`

## 八、已知問題與注意事項

### Excel ↔ Linux mount 同步（已不適用）
- 切到 JSON 後此問題消失 ✓

### LibreOffice 鎖檔（已不適用）
- 切到 JSON 後不用 recalc.py ✓

- 走「主表查詢」、找不到標 `(待補)`
- 累積 4-6 週後主表會夠完整

### PT 視覺辨識最耗 token
- 每週 PT 約佔總 token 60%
- 是否優化：未來再說（v5）

### index.html 拖檔可能無效（本機開啟用）
- 某些瀏覽器/環境拖整個資料夾失敗
- 解法：用「📂 選擇 data 資料夾」按鈕（最穩定）

### GitHub Pages 部署延遲
- git push 後 ~1-2 分鐘才會更新
- 如果同事看到的還是舊版、提示他們強制重整（Ctrl+F5）
